需要重新把以前练习过的思路重新拿起来了。仍旧是以soulmachine的模板为基础进行练习和复习，不过这次的难度需要加大一些。

1. 数组

1.1. 26. Remove Duplicates from Sorted Array

用个变量index来标记当前已经存储下来的数据的位置

1.2. 80. Remove Duplicates from Sorted Array II

基本和上一题一样，不过注意是可以重复两次，所以不用和当前的位置判断，和前一个位置判断就可以了。

1.3. 33. Search in Rotated Sorted Array

旋转数组
利用二分查找判断哪一段是有序的，然后进行分而治之。这个旋转数组是不重复的。
用闭区间的方式比较好区分些。

1.4. 81. Search in Rotated Sorted Array II

这个题目相对会困难一些，包含可能重复的元素，在分治的时候要注意区分条件，因为nums[left] <= nums[mid]分为小于和等于两种情况。
划分时[left, mid], [mid + 1, right]

1.5. 4. Median of Two Sorted Arrays

很经典的题目，第k大的数。

1.6. 128. Longest Consecutive Sequence

确定数组中可以组成的最长连续数列
要求时间复杂度为O(n)，那就必须用哈希了。
用哈希表表示查找每个元素，以及是否被使用过，如果没有被使用，则左右扩张。

1.7. 1. Two Sum

还是比较容易错的，可能数组中有重复的元素，哈希表中只保留了最后一次插入的。

1.8. 15. 3Sum

求三个数的和为给定值的组合，注意可能有重复的元素，但不能包含重复解，因此需要把每个元素重复的位置都给跳过。
需要先排序再夹逼，本题时间复杂度为O^2，推广后时间复杂度基本为O(max{n\log n, O^{k - 1}}。

如果要进一步优化的话，可能会复杂一些。

1.9. 16. 3Sum Closest

返回和与给定值差距最小的三个数
排序夹逼即可

1.10. 18. 4Sum

排序后左右夹逼，时间复杂度O(n^3)，可以考虑进一步优化，用hash

1.11. 27. Remove Element

1.12. 31. Next Permutation

建议复习一下组合数学的一些东西，当一个结论扩充。

1.13. 60. Permutation Sequence

利用康托编码的思路，需要推导公式。

同样的思路完全可以再简化一下写法
[LeetCode] Permutation Sequence 序列排序

1.14. 36. Valid Sudoku

模拟即可

1.15. 42. Trapping Rain Water

左右遍历找到每个元素左右的最高值

1.16. 48. Rotate Image

旋转图像，其实就是找规律

1.17. 66 Plus One

用数组表示的高精度加法

1.18. 70. Climbing Stairs

简单的斐波那契数列

1.19. 89. Gray Code

比较常规的思路，对称递归生成

也可以利用数学公式直接生成

1.20. 73. Set Matrix Zeroes

将有0的行和列全部置为0

方法一、空间复杂度$\mathcal{O} (n+m)$

方法二、空间复杂度$\mathcal{O}(1)$

1.21. 134. Gas Station

方法一、模拟，时间复杂度$\mathcal{O}(n^2)$

方法二、时间复杂度$\mathcal{O}(n)$，双变量标记即可
[LeetCode] Gas Station 加油站问题

1.22. 135. Candy

分糖游戏，更高rate的小朋友要比周边的更多，并且每个人至少一个糖

方法一、分两边进行扫描，统计增量即可

备忘录法，不错的思路，典型的递归

1.23. 136. Single Number

查找出现一次的数字，其他都出现2次

1.24. 137. Single Number II

查找出现2次的数字，其他都出现3次

方法一、用二进制模拟三进制，用one记录到当前元素位置，二进制出现1次的有哪些二进制位；用two表示出现2次的二进制位；当one和two都出现1时，该位的二进制出现了三次，即可清零。

方法二、模拟，统计对应位上出现次数为1的二进制即可

2. 链表

2.1. 2. Add Two Numbers

利用哨兵节点很容易完成

2.2. 206. Reverse Linked List

单纯的翻转链表，链接节点处理时有个规律，要取下该节点时肯定有两个操作，标记next成员的源和目的地

插入的时候也肯定有两个操作。所以要牢记这取下的节点和插入的地方的操作。

可以更为精简化，这个方法比较简洁一些

标准头插法

2.3. 92. Reverse Linked List II

用头插法就能简化问题，代码非常的简洁和直观

如果用翻转链表简化版的方法来处理的话，会比较容易错

2.4. 86. Partition List

2.5. 83. Remove Duplicates from Sorted List

2.6. 82. Remove Duplicates from Sorted List II

移除重复出现的数据

递归即可

2.7. 61. Rotate List

循环右移链表

首尾拼接再剪短即可

2.8. 19. Remove Nth Node From End of List

移除倒数第k个元素

双指针即可

2.9. 24. Swap Nodes in Pairs

2.10. 25. Reverse Nodes in k-Group

递归

2.11. 138. Copy List with Random Pointer

很经典的题目

2.12. 141. Linked List Cycle

2.13. 142. Linked List Cycle II

找到循环链表的入口点

注意下如何推导公式，慢慢要往更高抽象层次走，不仅仅是总结规律

2.14. 143. Reorder List

重排序链表

基本就是快慢指针找到链表中点再反转链接和拼接，仍旧应该思路很清晰
